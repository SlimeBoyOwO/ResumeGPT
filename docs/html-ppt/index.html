01 · Project Overview

研究背景与项目简介

研究背景

传统简历筛选依赖人工审阅，面临效率低下、主观偏差和难以规模化等问题。随着大语言模型技术的发展，利用 AI 技术实现简历的自动化审阅、语义匹配和多维度评分成为可能。

核心目标

  • 自动解析简历，提取结构化信息（NER）
  • 基于语义的简历-JD 匹配（RAG 粗筛）
  • 多专家 Agent 协同评分（精筛）
  • 可解释的结构化评分报告

03 · NER Model Architecture

NER 实体提取 · 模型架构

Encoder: chinese-roberta-wwm-ext

  • 基于 RoBERTa 的中文预训练模型
  • 全词遮蔽（Whole Word Masking）策略
  • 输出 768 维隐藏状态向量
  • 最大序列长度 512 tokens
  • 通过微调适配简历 NER 任务

Decoder: GlobalPointer

  • 替代传统 CRF，直接预测实体 span
  • 4D logits: [batch, 18, seq, seq]
  • 位置 (i,j) = token i..j 构成实体的得分
  • 旋转位置编码（RoPE）注入位置信息
  • 上三角掩码确保 start ≤ end
  • 缩放因子 1/√(head_size=64)

18 ENTITY CLASSES

姓名性别出生日期 电话最高学历籍贯 现居城市政治面貌大学 雇主职位工作内容 项目名称项目职责学位 毕业时间工作时间段项目时间段

03 · NER Training & Results

训练过程与实验结果

数据与预处理

  • 2000 份合成简历 PDF + JSON 标注
  • pdfplumber 解析 PDF 文本
  • RoBERTa 分词器编码（max_length=512）
  • 字符级 span → token 偏移量对齐
  • 标签矩阵：[18, seq_len, seq_len]
  • 训练集 : 验证集 = 9 : 1

训练策略

  • AdamW 优化器 · lr = 2e-5
  • 线性预热（10% 步数）+ 衰减
  • batch_size = 16 · 15 epochs
  • 混合精度训练（AMP + GradScaler）
  • 梯度裁剪 max_norm = 1.0
  • Early stopping patience = 3

FGM 对抗训练

  • Fast Gradient Method 嵌入扰动
  • 正常前向 + 反向后执行攻击
  • 扰动词嵌入 → 二次前向+反向
  • 恢复原始嵌入 → 更新参数
  • 提升模型对输入噪声的鲁棒性

推理引擎

  • 滑动窗口：chunk=450, overlap=50
  • 重叠区域实体去重（category, start, end）
  • 嵌套 JSON 重建：按位置分组
  • 相同 schema 块连续实体自动归组
  • 离线加载，HF Hub 完全禁用

03 · PII Masking Pipeline

PII 脱敏保护机制

在调用大语言模型前，自动替换敏感信息；LLM 返回后自动还原真实数据，实现隐私保护

01

NER 提取

BERT NER 优先提取
姓名和电话号码

02

PII 替换

姓名 → "张三"
电话 → "10086114514"

03

LLM 提取

脱敏文本 + 脱敏 NER hint
发送给 DeepSeek LLM

04

PII 还原

LLM 输出中的占位符
替换回真实值

设计要点

  • LLM 全程无法接触真实 PII 数据
  • NER 本地推理，敏感信息不出服务器
  • 占位符选择避免与真实数据冲突
  • 文本和 NER hint 双重脱敏

应用场景

  • 简历上传时的 LLM 兜底提取
  • 专家评判前的简历数据处理
  • 保障 GDPR/个人信息保护合规
  • 防止 LLM 训练数据泄露风险

04 · RAG + Two-Stage Reranking

初筛 · RAG + 双阶段重排序

向量化入库流程

  • 从 NER JSON 提取候选人关键信息
  • 学历等级映射（6 档）：中专=0 / 高中=1 / 大专=2 / 本科=3 / 学士/硕士=4 / 博士=5
  • 拼接工作经历（职位+内容）和项目经历（项目名+职责）为语义文本
  • m3e-base 编码为 768 维向量
  • Upsert 到 ChromaDB（cosine 距离）
  • LLM 自动补全学历：若 NER 学历字段为空，LLM 根据教育经历自动推断最高学历

自动匹配触发

  • 简历入库 → 匹配所有已有 JD
  • JD 入库 → 匹配所有已有简历
  • ≥85 分自动触发精筛流程
SCORING FORMULA
semantic = max(0, (1 - cos_dist)) × 100
hard_score（基于学历等级连续映射）:
hard_score = (edu_val / 5) × 40 + 60
• 博士(5) → 100   硕士(4) → 92
• 本科(3) → 84   大专(2) → 76
• 高中(1) → 68   中专(0) → 60
total = 0.3 × hard + 0.7 × semantic

双阶段重排序原理

  • 第一阶段：语义相似度召回（70% 权重）—— m3e-base 编码余弦距离
  • 第二阶段：硬性规则重排（30% 权重）—— 学历等级连续映射（6 档线性评分）
  • 学历自动补全：LLM 辅助提取时自动根据教育经历推断最高学历，确保 hard_score 不为空

05 · DAG Workflow Engine

精筛 · 大模型 DAG 工作流

DAG 拓扑排序执行引擎

  • Kahn 算法 BFS 确定执行顺序
  • 入度为 0 的节点可并行执行
  • 使用 asyncio.gather 并发调度
  • 每完成一个节点，下游节点入度 -1
  • 入度归零 → 入队执行

05 · Expert Prompt Tuning

专家评测标准 · Prompt Tuning

每个专家节点内置精细化的多维度评分标准，通过 Prompt Tuning 微调 system_prompt 实现评分一致性与可解释性

• 业务理解深度（0-30）

JD 逐条比对 · 满分 100

HR 综合评估专家

• 职业稳定性（0-25）· 沟通协作（0-25）
• 发展潜力（0-25）· 风险评估（0-25）

4 维度 · 满分 100
专家 system_prompt 多维度评分标准设计

Prompt Tuning 核心设计原则

  • 每个维度独立评分，禁止先想总分再拆分
  • 必须引用简历原文作为评分证据
  • 信息缺失时在对应档位低分段评分
  • 各维度含 4 档评分标准（表格量化）
  • 总分 = 各维度得分之和（不可自行调整）
