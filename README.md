# 基于大模型模型的简历审阅系统 - 个人毕设

## 一、项目简介

本项目是一个基于大模型模型的简历审阅系统，旨在帮助企业和求职者提高简历审阅的效率和准确性。系统采用大模型模型，通过训练大量简历数据，实现简历的自动审阅、推荐与打分功能。

**搭建的工作流是这样的**：

1.  首先pdf分析简历，解析文本content，如果是别的格式可以用ocr或者word解析，
2.  训练了一个f1 score为99的NLP，NER实体提取模型，可以初步提取关键信息，
3.  然后保存到数据库用于筛选等操作，之后使用向量数据库存储，
4.  企业用户可以发布JD，让JD与关键信息的关键部分匹配（如岗位，项目经历，工作内容等），
5.  在向量中算出语义相近度，进行粗匹配。
6.  被粗匹配的简历将会进入ToT工作流，
7.  JD可以自定义选择“简历分析专家”或者让大模型根据 JD 描述自动选择需要分析的专家和对应的评分权重（类似于MoE，用路由Agent选择激活的专家），
8.  每一个专家都有自己的分析侧重内容和专属的system prompt，他们会各自输出格式化内容并给出对应的评分，
9.  如果需要深度分析，还可以额外在A，B专家分析完毕后，让C专家结合他们进行如“软技能评分”的操作，最终让一个综合专家结合所有专家进行总结。(ToT工作流)
10. 根据权重计算最后得分。企业可以查看发布的JD的简历评分排行榜，从而实现最终的简历匹配和评分。

## 二、初步设想技术栈

- 前端：Vue3，Typescript，Tailwind CSS，Pinia，Vite，pnpm管理
- 后端：Python，FastAPI，uv管理
- 数据库：MySQL，ChromaDB
- 大模型模型：WebLLM
- 辅助技术（先验证再选用）：BERT模型，Embedding，RAG，Sentence-BERT，Cross-Encoder
- 架构理念：Agent方式的MoE混合专家模型，ToT 思维树调用模型

## 五、如何使用

- 后端：进入虚拟环境后`uvicorn main:app --reload`
- 前端：`pnpm run dev`

- 管理员账号：admin, admin123（实测哈希解析有问题，可以自己搞个用户之后，在数据库设定为admin）
- 用户账号：用账密在`localhost:5173`注册即可
